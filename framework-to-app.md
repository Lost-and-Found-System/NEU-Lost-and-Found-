# Framework-to-App Mapping
## Overview
This section maps the chosen KM framework (SECI Model) to the actual features of NEU Found Hub. Each SECI phase is linked to specific app behaviors, ensuring the system qualifies as a Knowledge Management platform rather than a generic CRUD application.

### Socialization (Tacit → Tacit)
- **App Features:** Comment threads, threaded replies, real-time notifications.  
- **Description:** Tacit knowledge is exchanged directly between community members. A student might say, *“I saw this near the south gate.”* Replies allow follow-up questions, while real-time notifications ensure no message is missed.
- **Rationale:** Socialization occurs when personal recollections and observations are shared informally. The app replicates this dialogue digitally, enabling immediate peer-to-peer knowledge exchange.

### Externalization (Tacit → Explicit)
- **App Features:** Item submission form, Gemini AI image analysis, Cloudinary image upload.
- **Description:** Tacit knowledge (e.g., *“I found this wallet in Building 3”*) is articulated into explicit records. A student photographs the item, Gemini AI generates a structured title and category, and the form submission creates a searchable database entry.
- **Rationale:** Externalization transforms individual observations into explicit, structured knowledge that can be stored, retrieved, and reused.

### Combination (Explicit → Explicit)
- **App Features:** Search, FilterBar (tabs + category pills), Resolved History archive, admin moderation.  
- **Description:** Explicit records are aggregated and organized. Search and filters allow users to combine item data with taxonomy categories. The Resolved History page consolidates resolved cases into an institutional archive.
- **Rationale:** Combination integrates multiple explicit records into structured knowledge, enabling organizational learning and efficient retrieval.

### Internalization (Explicit → Tacit)
- **App Features:** Intro.js onboarding tour, Resolved History browsing.
- **Description:** Explicit features and records are absorbed into tacit user understanding. The guided tour converts platform features into tacit knowledge through experience. Browsing resolved cases teaches students how items are described, how quickly they are recovered, and what practices are effective.
- **Rationale:** Internalization ensures that explicit knowledge becomes part of the user’s intuition and future behavior, strengthening community KM practices.

## KM Rationale
This mapping demonstrates that NEU Found Hub implements all four SECI phases:
- **Socialization:** Tacit knowledge shared via feed, comments, and notifications.
- **Externalization:** Tacit knowledge articulated through item posts, categorization, and image uploads.
- **Combination:** Explicit knowledge organized via search, filters, archives, and moderation.  
- **Internalization:** Explicit records absorbed into tacit user practice through onboarding and browsing.  

By aligning features with SECI, the app proves itself as a Knowledge Management system, not just a CRUD database.
