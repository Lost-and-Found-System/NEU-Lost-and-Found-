# Test Case TC-003

| Field | Value |
|-------|-------|
| **Test ID** | TC-003 |
| **Feature** | Resolving an issue |
| **Scenario** | Post owner marks their lost item as resolved |
| **Steps** | 1. User creates a lost item post<br> 2. User clicks the post and click the "Mark resolved" button on their own post. <br> 3. A message will appear " Item marked as resolved!" <br> 4. Confirm resolution |
| **Expected Result** | Post status changes to "Resolved". |
| **Actual Result** | After clicking "Mark resolved" button, confirmation dialog appeared. User confirmed. Success message "Item marked as resolved!" displayed. Post status changed from "Lost" or "Found" to "Resolved" on the post card and detail page. |
| **Status** | Pass  |
| **Notes** | Post is still visible in the feed of the user and resolved posts by the user is in the "Resolved" tab. |
