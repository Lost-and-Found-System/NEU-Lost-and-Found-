# Test Case TC-002

| Field | Value |
|-------|-------|
| **Test ID** | TC-002 |
| **Feature** | Commenting on a post + Notifications |
| **Scenario** | User comments on another user's post; owner receives notification |
| **Steps** | 1. User A logs in and creates a post <br> 2. User B logs in <br> 3. User B navigates to User A's post <br> 4. User B adds comment: "hi" <br> 5. Click "+" button to add comment <br> 6. User A checks notifications |
| **Expected Result** | Comment appears under post; User A receives a notification: "User B commented on your post" |
| **Actual Result** | Comment "hi" displayed under User A's post with User B's name and timestamp. User A's notification received a banner (1). Clicking notification opened the post and highlighted the new comment. Notification message: "New comment from User B: 'hi'". |
| **Status** | Pass |
| **Notes** | Notifications page displays all comment activities with timestamps (e.g., "New comment on item: hi" dated May 4, 2026, 12:26 PM). Each notification has a checkbox (possibly for batch actions). Comment appears immediately on the post card under the relevant item. |
