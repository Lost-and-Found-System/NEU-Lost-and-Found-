# Test Case TC-005

| Field | Value |
|-------|-------|
| **Test ID** | TC-005 |
| **Feature** | Filtering by lost / found + categories |
| **Scenario** | User filters posts by "Lost" or "Found" and categories, such as electronics, clothings, ID/cards, jewelries, and others |
| **Steps** | 1. On main feed, click the filter found or lost <br> 2. Click any category filter |
| **Expected Result** | Only posts that are filtered will be visible |
| **Actual Result** | Selected "Lost" filter + "electronics" category. Feed displayed only posts that were both lost and electronics category. Total of 3 resolved posts appeared. Tested another combination: "Lost" + "others" showed 2 resolved posts and 1 unresolved. All non-matching posts were hidden. |
| **Status** | Pass |
| **Notes** | Filters use AND logic (lost/found + category). Both resolved and unresolved posts appear within filters unless "Resolved" tab is selected separately. Category options tested: electronics, clothing, ID/cards, jewelry, others – all working. Clearing filters restores full feed.M |
